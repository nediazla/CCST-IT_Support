# **Módulo 6: Herramientas de trabajo**

## 1. Software de acceso remoto

El técnico de soporte utiliza herramientas de **acceso remoto** para conectarse a los dispositivos de los usuarios y realizar tareas de diagnóstico o mantenimiento a distancia.

---

### 1.1 Tipos de software de acceso remoto

| Herramienta | Descripción | Sistema compatible |
|--------------|-------------|--------------------|
| **Escritorio remoto (RDP)** | Incluido en Windows, permite controlar equipos mediante IP o nombre de host. | Windows |
| **Asistencia rápida / Quick Assist** | Herramienta de soporte remoto incluida en Windows 10/11. | Windows |
| **Cisco Webex / Teams / Zoom** | Permiten compartir pantalla y controlar sesiones remotas de soporte. | Multiplataforma |
| **VNC (Virtual Network Computing)** | Protocolo multiplataforma para acceso remoto por red local o Internet. | Windows, Linux, macOS |
| **TeamViewer / AnyDesk** | Aplicaciones de terceros para conexión remota segura mediante ID y contraseña temporal. | Multiplataforma |

**Buenas prácticas:**
- Solicitar permiso del usuario antes de tomar control.  
- Cerrar sesión al finalizar.  
- No transferir datos personales sin autorización.  
- Usar autenticación de dos factores si la herramienta lo permite.

**Ejemplo de conexión RDP:**
```cmd
mstsc /v:192.168.1.100
```

**Linux/macOS:**
```bash
rdesktop 192.168.1.100
```

## 2. Herramientas de solución de problemas

El soporte técnico requiere utilizar software especializado para **diagnóstico, monitoreo y documentación** de incidentes.

---

### 2.1 Uso de la Inteligencia Artificial (IA) en soporte técnico

La IA puede ayudar a los técnicos a investigar y resolver problemas más rápidamente, siempre respetando la privacidad y la ética.

| Tipo de IA | Descripción | Ejemplo |
|-------------|-------------|----------|
| **Predictiva** | Analiza datos históricos para anticipar fallos. | Predicción de fallos de hardware. |
| **Generativa** | Crea texto o soluciones basadas en lenguaje natural. | Asistentes como ChatGPT o Copilot. |

**Usos comunes:**
- Redacción de respuestas automáticas o sugerencias de solución.  
- Búsqueda guiada en bases de conocimiento.  
- Creación de documentación o guías.  

**Limitaciones y consideraciones éticas:**
- La IA no reemplaza la validación humana.  
- Puede generar errores si no se verifican las fuentes.  
- Nunca debe usarse para manejar información confidencial sin consentimiento.  

**Riesgos de privacidad y seguridad:**
- Fugas de datos sensibles al introducir información privada en sistemas externos.  
- Necesidad de anonimizar datos antes de su análisis por IA.  

---

### 2.2 Motores de búsqueda

Los **motores de búsqueda** (Google, Bing, DuckDuckGo) son herramientas básicas para localizar información técnica.

**Consejos de uso:**
- Usar palabras clave específicas: “Windows 11 error 0x80070005 solución”.  
- Incluir nombre de herramienta o versión.  
- Evitar copiar código sin entenderlo.  

**Operadores útiles:**
| Operador | Ejemplo | Función |
|-----------|----------|----------|
| `site:` | `site:microsoft.com BitLocker` | Busca solo en un sitio específico. |
| `filetype:` | `filetype:pdf Cisco VPN` | Busca archivos por tipo. |
| `" "` | `"error de impresora 0x00000709"` | Búsqueda exacta. |

---

### 2.3 Foros técnicos y comunidades

Los foros permiten encontrar soluciones verificadas y discutir problemas técnicos con expertos.

**Ejemplos:**
- Microsoft Learn Community.  
- Cisco Community.  
- Stack Overflow / SuperUser.  
- Reddit r/sysadmin o r/linux.  

**Buenas prácticas:**
- Buscar antes de preguntar.  
- Describir el problema con detalle (versión, sistema, síntomas).  
- Agradecer y compartir la solución final.  

---

### 2.4 Artículos de base de conocimiento (KB)

Las **bases de conocimiento (Knowledge Base)** contienen documentación interna y pública de soluciones comprobadas.

**Fuentes oficiales:**
- Microsoft KB Articles.  
- Cisco Support Documentation.  
- Apple Support Library.  
- Base de conocimiento interna de la empresa.

**Estructura ideal de un artículo técnico:**
1. Título claro.  
2. Descripción del problema.  
3. Causa raíz.  
4. Solución paso a paso.  
5. Referencias y versiones afectadas.  

**Ejemplo:**  
> **KB-0231: Error de conexión VPN 412**  
> **Causa:** Firewall bloquea protocolo GRE.  
> **Solución:** Permitir tráfico GRE en router o firewall.  
> **Verificado en:** Windows 11, Cisco AnyConnect 5.x

## 3. Documentación interna y registro de hallazgos

El técnico debe documentar cada intervención de manera que **otros puedan reproducir la solución**.

**Ejemplo de estructura de documentación interna:**

```
Ticket ID: #1452
Fecha: 08/11/2025
Usuario: Juan Pérez
Problema: No puede imprimir en red
Acciones: Reiniciado spooler, reinstalado controlador HP.
Resultado: Funciona correctamente.
Responsable: Nelson
```

**Consejos para documentación efectiva:**
- Escribir con lenguaje claro y objetivo.  
- Incluir comandos ejecutados y resultados.  
- Evitar opiniones o suposiciones sin evidencia.  
- Actualizar la base de conocimiento con los hallazgos.  

---

## 4. Ejemplo práctico

**Situación:** El usuario no puede conectarse por escritorio remoto.  
**Diagnóstico:** Servicio RDP detenido en el equipo remoto.  
**Solución:** Reiniciar servicio con PowerShell remoto:  
```powershell
Get-Service TermService | Start-Service
```
**Resultado:** Conexión establecida.  
**Documentación:** Ticket actualizado y cerrado.

## 5. Preguntas tipo examen

1. ¿Cuál de las siguientes herramientas permite controlar un equipo Windows de manera nativa?  
   A. TeamViewer  
   B. Escritorio remoto (RDP)  
   C. Cisco Webex  
   D. AnyDesk  
   **Respuesta:** B  

2. ¿Qué tipo de IA puede generar respuestas en lenguaje natural?  
   A. Predictiva  
   B. Generativa  
   C. Analítica  
   D. Descriptiva  
   **Respuesta:** B  

3. ¿Qué operador de búsqueda permite limitar resultados a un dominio específico?  
   A. `filetype:`  
   B. `inurl:`  
   C. `site:`  
   D. `domain:`  
   **Respuesta:** C  

4. ¿Qué debe contener un artículo de base de conocimiento?  
   A. Opiniones personales del técnico.  
   B. Pasos de solución y causas del problema.  
   C. Conversaciones de chat con el usuario.  
   D. Información confidencial del cliente.  
   **Respuesta:** B  

5. ¿Qué precaución se debe tener al usar IA para soporte técnico?  
   A. Introducir toda la información confidencial.  
   B. Omitir verificación humana.  
   C. Verificar la precisión y proteger la privacidad.  
   D. Confiar ciegamente en las sugerencias.

6. ¿Cuál es una práctica adecuada al usar software de acceso remoto?  
   A. Controlar el dispositivo sin permiso.  
   B. Mantener la sesión abierta indefinidamente.  
   C. Solicitar autorización y cerrar al finalizar.  
   D. Transferir datos personales sin notificar.  
   **Respuesta:** C  

7. ¿Qué estructura básica debe tener una documentación técnica interna?  
   A. Fecha, autor, problema, acciones y resultado.  
   B. Solo comandos ejecutados.  
   C. Comentarios personales del técnico.

   **Respuesta:** A
